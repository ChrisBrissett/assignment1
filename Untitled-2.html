Project Idea

Overview

The Project I would like to undertake and implement would be to design an app for use on the Google play and Apple store’s, which would replace the score sheets and cards for when playing the board game Cluedo. The game will be downloadable and useable to keep track of each game you play, making it a more effective method of being able to keep track of what cards you were dealt along with being able to make off options you have ruled out in the current game. The app will also randomly be able to select the cards which the players will need to guess to win the game along with randomly allocated what is leave evenly amongst all players.

Motivation

The motivation behind this idea, is myself being one of the many people worldwide who still enjoy the social experience of playing board games with family and friends, I find that constantly pieces of the game go missing but most significantly for the game of Cluedo is having to re-print pages upon pages of scoresheets to replace the once that are already all scribbled over. Whilst it is true that most board games can be played on some sort of gaming platform nowadays, trying to combine the accessibility to of mobile phones and the social togetherness of playing board games is the driving motivation behind this idea.

Description

The Application would be designed to require people to log onto their relevant app store and download the app. Upon opening the app players will be able to create or join a lobby which will allow others playing the same game to join and register them as apart of the current game, the person who creates the lobby will be allocated as the game host. The app will have the functionality to pre-allocate the cards that the players are required to guess to win the game.

Once this is done the game ‘Host’ will be choose a function on the app which will shuffle and deal the remaining cards to the number of players in the game, as would happen in any standard game of Cluedo. The app will be able to show a graphical image of the cards on the screen, personal to each player. What this will achieve is that as the game progresses each person using their own app will be able to minimize their cards and refer to them at any time, this would also be relevant when having to show another player a card of yours. Players will be able to select a card and in ‘Show’ mode, which is a mode specifically designed when requiring to show a card, they will be able to present their phone to another player, once the screen has been touched the card will appear face up for 3 seconds, allowing the other person to mark it on their scoresheet.

The scoresheet being the other operational part of the app designed to make the game easier to play, will allow the user to check boxes as per the cards they have but also ones that have seen by another player. There will also be a shading system for the scoresheet as well which would allow you mark off the guesses being made by other players. For anyone who has ever played Cluedo before would know the benefits for being able to reach a conclusion based on what is being guessed more often. The benefits of using the app especially for this part of the game will remove the ‘cubby houses’ of boxes people build so no one can see what they have written down.

As mentioned previously obviously there are already electronic versions of this game available on various platforms, with the app and its design the group of people playing will be able to choose at what level of interaction with the game would be required from the app. That being, choosing whether to only use the scoresheet or using the cards as well, all dependent on player preference. The app is not designed to take over the interaction in place between people playing but to allow a more streamline experience and decrease the chances of a table being flipped over in the living room.

Tools and Technoolgies

The tools and technologies required to implement this idea would be the use of PhoneGap by Adobe. The reasoning behind using this is that PhoneGap is a cross-platform developer for mobile apps. Other technologies to consider using would be Java for designing the app for Android operating systems or Objective-C for the Apple store. Along with the above, we would need a cloud server to store the information in the app and allowing players to be able to connect to the same lobby.

Skills Required

To be able to write and develop the proposed app we would need to have the skill in the above app development tools, along with the knowledge of how Object-Orientated programming functionality works to be able to develop the code needed to create the basis for the app to function. Luckily, for this app to be developed, the hardware is already in place as mobile phones and being able download apps from the app store is already easily accessible. The main challenge being faced with developing the app is gaining a comprehensive knowledge of the development tools listed above.

Outcome

Overall, I believe this project to be one that could be quite achievable. If we were to successfully develop the app the main issue the app is designed to alleviate will be achieved. This would remove the paper scoresheet requirement from the game. Whilst the app is giving users access to a digital version of the scoresheet and cards, it will not remove the social interaction surrounding playing the game. This will solve the original problem of running out the paper scoresheets, as well as any lost or misplaced pieces of the game.
